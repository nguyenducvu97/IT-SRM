# Multilingual Implementation Guide
## IT Service Request Management System

### Overview
This document describes the multilingual implementation supporting Vietnamese, English, and Korean languages in the IT Service Request Management System.

### Architecture

#### 1. Translation System Core
- **File**: `assets/js/translation.js`
- **Class**: `TranslationSystem`
- **Global Function**: `t(key, fallback)`

#### 2. Language Files Structure
```
assets/js/languages/
├── vi.js    # Vietnamese (default)
├── en.js    # English
└── ko.js    # Korean
```

#### 3. Translation Keys Structure
Each language file contains translation objects organized by category:
```javascript
const vi = {
    // Header & Navigation
    'app_title': 'IT Service Request Management',
    'system_subtitle': 'Hệ thống quản lý yêu cầu dịch vụ CNTT',
    
    // Authentication
    'login': 'Đăng nhập',
    'register': 'Đăng ký',
    // ... more keys
};
```

### Implementation Details

#### 1. Language Detection & Storage
- **Browser Language Detection**: Automatically detects browser language
- **Local Storage**: Persists user language preference
- **Fallback**: Defaults to Vietnamese if no preference stored

#### 2. Dynamic Language Switching
- **No Page Reload**: Language changes apply immediately
- **Event-Driven**: Uses custom events for component updates
- **Data Re-rendering**: Stored data re-renders with new translations

#### 3. HTML Integration
Elements use `data-translate` attributes:
```html
<button data-translate="login">Đăng nhập</button>
<span data-translate="system_subtitle">Hệ thống quản lý yêu cầu dịch vụ CNTT</span>
```

#### 4. JavaScript Integration
Use the global `t()` function:
```javascript
const translatedText = t('login', 'Login'); // with fallback
```

### Usage Guide

#### For Developers

##### Adding New Translation Keys
1. Add keys to all language files (`vi.js`, `en.js`, `ko.js`)
2. Use consistent key naming convention (snake_case)
3. Add `data-translate` attributes to HTML elements
4. Use `t()` function in JavaScript for dynamic content

##### Adding New Languages
1. Create new language file (e.g., `ja.js` for Japanese)
2. Add language option to switcher in HTML
3. Update `TranslationSystem` to load new language
4. Test all translation keys

##### Translating Dynamic Content
For content generated by JavaScript:
```javascript
// In display functions
element.innerHTML = t('status_' + status);

// In status/priority text functions
getStatusText(status) {
    return t(`status_${status}`) || status;
}
```

#### For Users

##### Language Switching
1. Click language dropdown in header
2. Select desired language (🇻🇳 Tiếng Việt, 🇺🇸 English, 🇰🇷 한국어)
3. Interface updates immediately
4. Preference saved for future visits
