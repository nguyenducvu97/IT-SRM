🔧 Test Search Functionality Fix

📋 Problem Fixed

Root Cause: Search event listener was added in constructor, but the search input element didn't exist at that time because it's inside the requestsPage which wasn't active yet.

Solution: Moved search event listener initialization to loadRequests() function where the element definitely exists.

✅ Changes Made

  1. Removed from constructor: Search event listener (line 437-485)
  2. Added to loadRequests(): Search event listener with duplicate check
  3. Duplicate prevention: Added data-search-initialized attribute
  4. Version update: app.js v=20260421-13 → v=20260421-14

🎯 How It Works Now

  1. User navigates to "Yêu cầu dịch vụ" page
  2. showPage('requests') is called
  3. loadRequests() is called
  4. Search input element is found (requestsPage is now active)
  5. Event listener is attached with duplicate check
  6. User types in search box → Debounce 500ms → loadRequests(1) called

🧪 Testing Steps
